Pattern 01
Two Pointer Technique
Reduces O(n²) brute force to O(n) by maintaining two indices
Core Idea

Place two pointers — often left and right — at opposite ends (or both at start) and move them based on a condition. The key insight: when the array is sorted, you know exactly which direction to move each pointer to get closer to the answer, eliminating the need for a nested loop.

Q1
Two Sum II — Input Array Is Sorted
Easy Two Pointer
+
Problem

Given a sorted array, find two numbers that add up to a target. Return their 1-indexed positions. You may not use the same element twice.

Input: [2, 7, 11, 15], target = 9 → Output: [1, 2]

Technique — Opposite-End Two Pointer

Because array is sorted: if arr[left] + arr[right] > target, the sum is too big → move right inward. If too small → move left outward. The sorted order guarantees convergence without missing any pair.

  • 1
    Set left = 0, right = n-1
  • 2
    Compute sum = arr[left] + arr[right]
  • 3
    If sum == target → return [left+1, right+1]
  • 4
    If sum < target → left++ (need bigger number)
  • 5
    If sum > target → right-- (need smaller number)
def two_sum(nums: list[int], target: int) -> list[int]:
    left, right = 0, len(nums) - 1
    while left < right:
        total = nums[left] + nums[right]
        if total == target:
            return [left + 1, right + 1]
        elif total < target:
            left += 1
        else:
            right -= 1
    return [-1, -1]  # no solution
TimeO(n)
SpaceO(1)
Q2
Remove Duplicates from Sorted Array
Easy Two Pointer
+
Problem

Given a sorted array, remove duplicates in-place and return the count of unique elements. Don't allocate extra array.

Input: [1,1,2,3,3,4] → Output: 4, array = [1,2,3,4,...]

Technique — Slow-Fast Pointer

slow pointer marks the "write position" for unique elements. fast scans ahead. When fast finds a new unique value (different from slow's position), write it to slow+1 and advance slow. This keeps only unique values packed at the front.

def remove_duplicates(nums: list[int]) -> int:
    if not nums:
        return 0
    slow = 0  # write pointer
    for fast in range(1, len(nums)):
        if nums[fast] != nums[slow]:
            slow += 1
            nums[slow] = nums[fast]  # write new unique
    return slow + 1
TimeO(n)
SpaceO(1)
Q3
3Sum — Find All Triplets that Sum to Zero
Medium Two Pointer
+
Problem

Given an array, find all unique triplets [a, b, c] such that a + b + c = 0. The solution set must not contain duplicate triplets.

Input: [-1,0,1,2,-1,-4] → Output: [[-1,-1,2],[-1,0,1]]

Technique — Sort + Fix One + Two Pointer

Sort the array first. Fix element at index i, then use two-pointer on the remaining right portion to find pairs that sum to -nums[i]. Skip duplicate values at each pointer position to avoid duplicate triplets.

  • 1
    Sort the array
  • 2
    For each i from 0 to n-3: skip if nums[i] == nums[i-1] (duplicate)
  • 3
    Set left = i+1, right = n-1, find pairs summing to -nums[i]
  • 4
    When pair found, skip duplicates from both ends before continuing
def three_sum(nums: list[int]) -> list[list[int]]:
    nums.sort()
    res = []
    for i in range(len(nums) - 2):
        if i > 0 and nums[i] == nums[i - 1]:
            continue  # skip duplicate i
        left, right = i + 1, len(nums) - 1
        while left < right:
            total = nums[i] + nums[left] + nums[right]
            if total == 0:
                res.append([nums[i], nums[left], nums[right]])
                while left < right and nums[left] == nums[left + 1]: left += 1
                while left < right and nums[right] == nums[right - 1]: right -= 1
                left += 1; right -= 1
            elif total < 0:
                left += 1
            else:
                right -= 1
    return res
TimeO(n²)
SpaceO(1) aux
Common Mistake

Forgetting to skip duplicates after finding a triplet leads to duplicate results. The while loops after adding a result handle this.

Q4
Container With Most Water
Medium Two Pointer
+
Problem

Given heights[] where each bar can form a wall, find two bars that together with x-axis forms a container holding the most water.

Water = min(height[l], height[r]) × (r - l)

Technique — Greedy Two Pointer

Always move the pointer with the shorter height inward. Why? The width is decreasing either way — the only hope of finding more water is to find a taller wall. Moving the taller side inward can only decrease area (width decreases, height can't increase beyond the current shorter bar).

def max_area(height: list[int]) -> int:
    left, right, max_water = 0, len(height) - 1, 0
    while left < right:
        water = min(height[left], height[right]) * (right - left)
        max_water = max(max_water, water)
        if height[left] < height[right]:
            left += 1
        else:
            right -= 1
    return max_water
TimeO(n)
SpaceO(1)
Q5
Valid Palindrome
Easy Two Pointer
+
Problem

Check if a string is a palindrome, considering only alphanumeric chars and ignoring case.

Technique — Two Pointer with Char Skipping

Start from both ends, skip non-alphanumeric characters, compare. This avoids creating a cleaned string (saves space).

def is_palindrome(s: str) -> bool:
    left, right = 0, len(s) - 1
    while left < right:
        while left < right and not s[left].isalnum(): left += 1
        while left < right and not s[right].isalnum(): right -= 1
        if s[left].lower() != s[right].lower():
            return False
        left += 1; right -= 1
    return True
TimeO(n)
SpaceO(1)

Pattern 02
Sliding Window
Extend right, shrink left — maintains a window satisfying a condition
Core Idea

Maintain a window [left, right] over the array. Expand right to include new elements. When the window violates a condition, shrink from left. The key: instead of computing the window from scratch each time, update it incrementally. Two variants: Fixed size (always k elements) and Variable size (expand/shrink based on a condition).

Q6
Maximum Average Subarray of Length K
Easy Sliding Window Fixed
+
Problem

Find the contiguous subarray of length k with the maximum average.

Input: [1,12,-5,-6,50,3], k=4 → Output: 12.75

Technique — Fixed Window: Slide by adding right, removing left

Build first window of k elements. Then slide: add nums[i] (new right) and subtract nums[i-k] (old left). O(1) update instead of O(k) recomputation.

def find_max_average(nums: list[int], k: int) -> float:
    window_sum = sum(nums[:k])  # first window
    max_sum = window_sum
    for i in range(k, len(nums)):
        window_sum += nums[i] - nums[i - k]  # slide
        max_sum = max(max_sum, window_sum)
    return max_sum / k
TimeO(n)
SpaceO(1)
Q7
Longest Substring Without Repeating Characters
Medium Sliding Window Variable
+
Problem

Given a string, find the length of the longest substring without repeating characters.

Input: "abcabcbb" → Output: 3 ("abc")

Technique — Variable Window + HashMap (char → last index)

Expand right. If s[right] already exists in the window, jump left past its last occurrence (don't just move left by 1 — use the stored index for O(1) shrink). Store char → index in a map.

def length_of_longest_substring(s: str) -> int:
    char_index = {}  # char -> last seen index
    max_len = left = 0
    for right, ch in enumerate(s):
        if ch in char_index:
            left = max(left, char_index[ch] + 1)  # jump left
        char_index[ch] = right
        max_len = max(max_len, right - left + 1)
    return max_len
TimeO(n)
SpaceO(min(m,n))
Key Insight

left = max(left, char_index[ch] + 1) — we take max because the stored index could be before the current left, meaning that char is already outside our window.

Q8
Minimum Size Subarray Sum
Medium Sliding Window Variable
+
Problem

Given an array of positive integers and target, find the minimal length subarray whose sum ≥ target. Return 0 if none.

Input: target=7, [2,3,1,2,4,3] → Output: 2 (subarray [4,3])

Technique — Shrink Window from Left when Condition Met

Expand right to accumulate sum. Once sum ≥ target, record length, then shrink from left (subtract nums[left], left++) while sum remains ≥ target. This finds the minimum length window satisfying the condition.

def min_sub_array_len(target: int, nums: list[int]) -> int:
    left = window_sum = 0
    min_len = float('inf')
    for right in range(len(nums)):
        window_sum += nums[right]
        while window_sum >= target:
            min_len = min(min_len, right - left + 1)
            window_sum -= nums[left]  # shrink from left
            left += 1
    return 0 if min_len == float('inf') else min_len
TimeO(n)
SpaceO(1)
Q9
Longest Repeating Character Replacement
Medium Sliding Window + Freq
+
Problem

You can replace at most k characters. Find the longest substring with the same letter after replacements.

Input: "AABABBA", k=1 → Output: 4

Technique — Window validity: (windowSize - maxFreq) ≤ k

A window is valid if the number of characters we need to replace = (window size - count of most frequent char) ≤ k. Track frequency of chars in window. When window becomes invalid, slide left by 1. The window only ever grows (we want maximum length).

def character_replacement(s: str, k: int) -> int:
    count = [0] * 26
    left = max_freq = res = 0
    for right in range(len(s)):
        count[ord(s[right]) - ord('A')] += 1
        max_freq = max(max_freq, count[ord(s[right]) - ord('A')])
        if (right - left + 1) - max_freq > k:  # invalid, shrink
            count[ord(s[left]) - ord('A')] -= 1
            left += 1
        res = max(res, right - left + 1)
    return res
TimeO(n)
SpaceO(1)

Pattern 03
Prefix Sum
Precompute cumulative sums to answer range queries in O(1)
Core Idea

Build array prefix[i] = sum(arr[0..i-1]). Then sum of any range [l, r] = prefix[r+1] - prefix[l]. This converts O(n) per query to O(1). For subarray sum problems: use a dictionary storing {prefix_sum: count} to find subarrays with a given sum in O(n).

Q10
Range Sum Query — Immutable
Easy Prefix Sum
+
Problem

Given an array, handle multiple queries of form sumRange(l, r) — return sum of elements from index l to r inclusive.

Technique — Build prefix array once, answer in O(1)

prefix[0] = 0, prefix[i] = prefix[i-1] + nums[i-1]. Then sumRange(l,r) = prefix[r+1] - prefix[l]. One-time O(n) preprocessing makes each query O(1).

class NumArray:
    def __init__(self, nums: list[int]):
        self.prefix = [0] * (len(nums) + 1)
        for i, n in enumerate(nums):
            self.prefix[i + 1] = self.prefix[i] + n

    def sum_range(self, left: int, right: int) -> int:
        return self.prefix[right + 1] - self.prefix[left]
BuildO(n)
QueryO(1)
SpaceO(n)
Q11
Subarray Sum Equals K
Medium Prefix Sum + HashMap
+
Problem

Count the total number of subarrays whose sum equals k.

Input: [1,1,1], k=2 → Output: 2

Technique — Prefix Sum + "Complement Lookup" in HashMap

If prefix sum at index r is S, we want to know: how many earlier indices l have prefix sum = S - k? Because then subarray [l, r] has sum k. Store prefix_sum → count in a map as we scan. Init map with {0: 1} to handle subarrays starting from index 0.

from collections import defaultdict

def subarray_sum(nums: list[int], k: int) -> int:
    count_map = defaultdict(int)
    count_map[0] = 1  # empty subarray has sum 0
    count = prefix_sum = 0
    for num in nums:
        prefix_sum += num
        count += count_map[prefix_sum - k]  # how many valid start points?
        count_map[prefix_sum] += 1
    return count
TimeO(n)
SpaceO(n)
The Core Formula

sum(l, r) = prefix_sum[r] - prefix_sum[l-1] = k → prefix_sum[l-1] = prefix_sum[r] - k. We look up how many times we've seen prefix_sum - k before in our dictionary.

Q12
Product of Array Except Self
Medium Prefix + Suffix Product
+
Problem

Return array where each element is the product of all other elements. No division, O(n) time, O(1) extra space.

Input: [1,2,3,4] → Output: [24,12,8,6]

Technique — Left pass × Right pass in output array

First pass (left to right): fill output[i] with product of all elements to the left. Second pass (right to left): multiply by running right product. Total: 2 passes, O(1) extra space.

def product_except_self(nums: list[int]) -> list[int]:
    n = len(nums)
    res = [1] * n
    for i in range(1, n):  # left products
        res[i] = res[i - 1] * nums[i - 1]
    right = 1
    for i in range(n - 1, -1, -1):  # multiply by right products
        res[i] *= right
        right *= nums[i]
    return res
TimeO(n)
SpaceO(1) aux

Q13
Two Sum (Unsorted)
Easy HashMap
+
Problem

Find indices of two numbers that add up to target in an unsorted array.

Technique — Store (value → index), look for complement

For each number, check if its complement (target - num) is already in the map. If yes → found the pair. If no → store current (num → index) in map. Single pass, no sorting needed.

def two_sum(nums: list[int], target: int) -> list[int]:
    seen = {}  # value -> index
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i
    return []
TimeO(n)
SpaceO(n)
Q14
Group Anagrams
Medium HashMap + Sorted Key
+
Problem

Group strings that are anagrams of each other.

Input: ["eat","tea","tan","ate","nat","bat"] → Output: [["bat"],["nat","tan"],["ate","eat","tea"]]

Technique — Canonical Key: sorted string as group key

Anagrams have the same sorted characters. Use Arrays.sort(chars) as the HashMap key. All anagrams map to the same key, grouping them automatically. Alternative: use char frequency array [26 counts] as key (avoids sort).

from collections import defaultdict

def group_anagrams(strs: list[str]) -> list[list[str]]:
    groups = defaultdict(list)
    for s in strs:
        key = tuple(sorted(s))  # canonical sorted key
        groups[key].append(s)
    return list(groups.values())
TimeO(n·k·log k)
SpaceO(n·k)
k= max str len
Q15
Longest Consecutive Sequence
Medium HashSet
+
Problem

Find the length of the longest consecutive elements sequence in an unsorted array. Must be O(n).

Input: [100,4,200,1,3,2] → Output: 4 (sequence [1,2,3,4])

Technique — Only start counting from sequence beginners

Put all numbers in a HashSet. For each number, only start counting if (n-1) is NOT in the set — meaning n is the start of a sequence. Then count n+1, n+2... while they exist in set. This ensures each number is visited at most twice: O(n).

def longest_consecutive(nums: list[int]) -> int:
    num_set = set(nums)
    max_len = 0
    for n in num_set:
        if (n - 1) not in num_set:  # sequence start
            length = 1
            while (n + length) in num_set:
                length += 1
            max_len = max(max_len, length)
    return max_len
TimeO(n)
SpaceO(n)
Q16
Top K Frequent Elements
Medium HashMap + Bucket Sort
+
Problem

Given an array, return the k most frequent elements. Must be better than O(n log n).

Input: [1,1,1,2,2,3], k=2 → Output: [1,2]

Technique — Bucket Sort by frequency (O(n) approach)

Count frequencies. Create buckets array of size n+1 where buckets[freq] = list of numbers with that frequency. Scan buckets from high freq to low, collect k elements. This avoids sorting (O(n log n)) and beats the heap approach for this specific problem.

from collections import Counter

def top_k_frequent(nums: list[int], k: int) -> list[int]:
    freq = Counter(nums)
    buckets = [[] for _ in range(len(nums) + 1)]
    for num, f in freq.items():
        buckets[f].append(num)  # group by frequency
    res = []
    for i in range(len(buckets) - 1, 0, -1):
        for num in buckets[i]:
            res.append(num)
            if len(res) == k:
                return res
    return res
TimeO(n)
SpaceO(n)

Pattern 05
Kadane's Algorithm & Subarray Extremes
Elegant DP-like approach for contiguous subarray max/min
Core Idea

At each index, ask: "Is it better to extend the current subarray, or start fresh?" Maintain currentMax = max(num, currentMax + num). If currentMax goes negative, restart from current element. Track global max separately.

Q17
Maximum Subarray (Kadane's Classic)
Medium Kadane's
+
Problem

Find the contiguous subarray with the largest sum.

Input: [-2,1,-3,4,-1,2,1,-5,4] → Output: 6 (subarray [4,-1,2,1])

Technique — Extend or Restart at each element

If the running sum becomes negative, it's a liability — starting fresh from the current element is always better. cur = max(num, cur + num) captures this decision in one line.

def max_sub_array(nums: list[int]) -> int:
    cur = max_sum = nums[0]
    for num in nums[1:]:
        cur = max(num, cur + num)
        max_sum = max(max_sum, cur)
    return max_sum
TimeO(n)
SpaceO(1)
Q18
Maximum Product Subarray
Medium Kadane's Variant
+
Problem

Find the contiguous subarray with the largest product.

Input: [2,3,-2,4] → Output: 6 ([2,3])

Technique — Track both curMax AND curMin (negatives flip sign)

Unlike sum, product has a twist: a large negative × another negative = large positive. So we must track the current minimum product too. When we encounter a negative, swap maxProd and minProd before multiplying, because the minimum becomes maximum and vice versa.

def max_product(nums: list[int]) -> int:
    result = cur_max = cur_min = nums[0]
    for num in nums[1:]:
        candidates = (num, cur_max * num, cur_min * num)
        cur_max, cur_min = max(candidates), min(candidates)
        result = max(result, cur_max)
    return result
TimeO(n)
SpaceO(1)
Key Difference from Sum Version

Computing all three candidates (num, cur_max*num, cur_min*num) at once before updating avoids using a stale value. Python's tuple unpacking makes this clean and safe in one line.

Q19
Best Time to Buy and Sell Stock
Easy Kadane's Spirit
+
Problem

Given daily prices, find the max profit from one buy-sell transaction (buy before sell).

Input: [7,1,5,3,6,4] → Output: 5 (buy at 1, sell at 6)

Technique — Track min price seen so far

As you scan, maintain the minimum price seen so far. At each day, potential profit = current price - minSoFar. Update global max profit. This is equivalent to Kadane's on the price differences array.

def max_profit(prices: list[int]) -> int:
    min_price = float('inf')
    max_profit = 0
    for price in prices:
        min_price = min(min_price, price)
        max_profit = max(max_profit, price - min_price)
    return max_profit
TimeO(n)
SpaceO(1)

Q20
Valid Anagram
Easy Freq Array
+
Problem

Determine if two strings are anagrams of each other.

Technique — One frequency array, add for s, subtract for t

Increment freq[c-'a'] for every char in s, decrement for every char in t. If all entries are 0 at the end, they're anagrams. O(1) space (fixed 26 slots).

def is_anagram(s: str, t: str) -> bool:
    if len(s) != len(t):
        return False
    freq = [0] * 26
    for c in s: freq[ord(c) - ord('a')] += 1
    for c in t: freq[ord(c) - ord('a')] -= 1
    return all(f == 0 for f in freq)
TimeO(n)
SpaceO(1)
Q21
Find All Anagrams in a String
Medium Sliding Window + Freq
+
Problem

Find all starting indices where a permutation (anagram) of pattern p exists in string s.

Input: s="cbaebabacd", p="abc" → Output: [0, 6]

Technique — Fixed window of size |p| + track "matches" count

Track how many chars currently have matching frequency (using a matches counter). When adding/removing a char from window, update matches. When matches == 26, window is an anagram. This avoids comparing full arrays each step.

def find_anagrams(s: str, p: str) -> list[int]:
    p_freq = [0] * 26
    s_freq = [0] * 26
    for c in p: p_freq[ord(c) - ord('a')] += 1
    matches = sum(1 for f in p_freq if f == 0)  # chars not in p already match
    res, left = [], 0
    for right in range(len(s)):
        r = ord(s[right]) - ord('a')
        s_freq[r] += 1
        if s_freq[r] == p_freq[r]: matches += 1
        elif s_freq[r] == p_freq[r] + 1: matches -= 1
        if right - left + 1 == len(p):
            if matches == 26: res.append(left)
            l = ord(s[left]) - ord('a')
            s_freq[l] -= 1
            if s_freq[l] == p_freq[l]: matches += 1
            elif s_freq[l] == p_freq[l] - 1: matches -= 1
            left += 1
    return res
TimeO(n)
SpaceO(1)
Q22
Longest Palindromic Substring
Medium Expand Around Center
+
Problem

Find the longest palindromic substring in a string.

Input: "babad" → Output: "bab" or "aba"

Technique — Expand Around Each Center (Odd + Even centers)

For each index, try expanding outward as long as chars match. Handle both odd-length (center = single char) and even-length (center = between two chars) palindromes. O(n) centers × O(n) expansion = O(n²), which beats O(n³) brute force.

def longest_palindrome(s: str) -> str:
    def expand(l: int, r: int) -> int:
        while l >= 0 and r < len(s) and s[l] == s[r]:
            l -= 1; r += 1
        return r - l - 1  # length of palindrome

    start, max_len = 0, 1
    for i in range(len(s)):
        odd  = expand(i, i)      # odd length
        even = expand(i, i + 1) # even length
        length = max(odd, even)
        if length > max_len:
            max_len = length
            start = i - (length - 1) // 2
    return s[start : start + max_len]
TimeO(n²)
SpaceO(1)
Manacher's Algorithm

There is an O(n) algorithm called Manacher's, but it's complex. For interviews, O(n²) expand-around-center is perfectly acceptable and much easier to explain.

Q23
Encode and Decode Strings
Medium String Encoding
+
Problem

Design an algorithm to encode a list of strings to a single string and decode it back. The encoded string is sent over the network.

Technique — Length-prefix encoding: "length#string"

Encode each string as len + "#" + str. Decode by reading length, jumping past "#", reading exactly that many chars. This handles strings containing any characters including "#".

def encode(strs: list[str]) -> str:
    return "".join(f"{len(s)}#{s}" for s in strs)

def decode(s: str) -> list[str]:
    res, i = [], 0
    while i < len(s):
        j = s.index('#', i)
        length = int(s[i:j])
        res.append(s[j + 1 : j + 1 + length])
        i = j + 1 + length
    return res
EncodeO(n)
DecodeO(n)
Q24
Minimum Window Substring
Hard Sliding Window + Two Freq Maps
+
Problem

Find the minimum window in s that contains all characters of t (with duplicates).

Input: s="ADOBECODEBANC", t="ABC" → Output: "BANC"

Technique — Variable window with "formed" count tracking

Use two frequency maps. Track formed = number of unique chars in t that are currently satisfied in window. Expand right until all of t is covered, then shrink left as much as possible while still covering t. Record minimum window each time all chars are satisfied.

from collections import Counter

def min_window(s: str, t: str) -> str:
    if not t: return ""
    t_map = Counter(t)
    w_map = {}
    required = len(t_map)
    formed = left = 0
    best = (float('inf'), 0, 0)  # (length, left, right)
    for right, ch in enumerate(s):
        w_map[ch] = w_map.get(ch, 0) + 1
        if ch in t_map and w_map[ch] == t_map[ch]:
            formed += 1
        while formed == required:
            if right - left + 1 < best[0]:
                best = (right - left + 1, left, right)
            lc = s[left]
            w_map[lc] -= 1
            if lc in t_map and w_map[lc] < t_map[lc]:
                formed -= 1
            left += 1
    return "" if best[0] == float('inf') else s[best[1] : best[2] + 1]
TimeO(|s| + |t|)
SpaceO(|t|)
Q25
Trapping Rain Water
Hard Two Pointer (Classic Hard)
+
Problem

Given heights of bars, compute how much rain water is trapped between them.

Input: [0,1,0,2,1,0,1,3,2,1,2,1] → Output: 6

Technique — Two Pointer with maxLeft/maxRight tracking

Water trapped at any index = min(maxLeft, maxRight) - height[i]. Use two pointers: always process the side with the smaller max height. If maxLeft ≤ maxRight, water at left = maxLeft - height[left] (we know maxRight is even larger). Advance the smaller side inward.

def trap(height: list[int]) -> int:
    left, right = 0, len(height) - 1
    max_left = max_right = water = 0
    while left < right:
        if height[left] <= height[right]:
            if height[left] >= max_left:
                max_left = height[left]
            else:
                water += max_left - height[left]
            left += 1
        else:
            if height[right] >= max_right:
                max_right = height[right]
            else:
                water += max_right - height[right]
            right -= 1
    return water
TimeO(n)
SpaceO(1)
Why this works

When height[left] ≤ height[right], maxRight is guaranteed to be ≥ height[right] ≥ height[left]. So the bottleneck is maxLeft, and water = maxLeft - height[left] is exact.
